## 🔍 What is NEAR MPC Network?

### Definition

**MPC (Multi-Party Computation):** A way for multiple parties to compute something together WITHOUT any party revealing their private data.

### How It Works

```
Traditional computation:
  User → Server → Result
  (Server sees everything)

MPC computation:
  User → [Node1, Node2, Node3, ...] → Result
  (No single node sees everything!)
```

### Example: Secret Addition

```
Alice has: 5
Bob has: 7

Goal: Compute 5 + 7 = 12
Without: Anyone knowing Alice has 5 or Bob has 7

MPC Solution:
  Alice splits 5 → [2, 3]
  Bob splits 7 → [4, 3]
  
  Node1 computes: 2 + 4 = 6
  Node2 computes: 3 + 3 = 6
  
  Result: 6 + 6 = 12
  But: Node1 doesn't know Alice's 5 or Bob's 7!
```

## 🏗️ Architecture: MPC as Delegator

### Traditional Delegator

```
User (alice.near)
    ↓ Sends signature
Delegator (delegator.near) - CENTRALIZED
    ↓ Verifies + Generates hash
    ↓ Calls contract
Smart Contract
```

**Problems:**
- Delegator sees alice.near
- Single point of failure
- Trust required

### MPC Delegator

```
User (alice.near)
    ↓ Sends signature (encrypted/sharded)
MPC Network (mpc.near) - DECENTRALIZED
    ├─ Node 1: Verifies partial data
    ├─ Node 2: Verifies partial data
    ├─ Node 3: Verifies partial data
    └─ Collective: Generates hash
    ↓ Calls contract (collectively)
Smart Contract
```

**Benefits:**
- No single node sees alice.near
- Decentralized
- Trustless

## 🚀 Implementation

### Smart Contract Changes

```rust
// Add MPC-specific registration method
pub fn register_via_mpc(
    &mut self,
    npub: String,
    commitment: String,
    nullifier: String,
    mpc_proof: MPCProof,
    mpc_signature: String,
) -> IdentityInfo {
    // 1. Verify MPC proof
    if !verify_mpc_proof(&mpc_proof) {
        panic!("Invalid MPC proof")
    }
    
    // 2. Verify MPC signature
    if !verify_mpc_signature(&mpc_signature) {
        panic!("Invalid MPC signature")
    }
    
    // 3. Verify commitment matches MPC computation
    if !verify_commitment_from_mpc(&commitment, &mpc_proof) {
        panic!("Commitment mismatch")
    }
    
    // 4. Store identity (same as before)
    let identity = IdentityInfo {
        npub,
        commitment: commitment.clone(),
        nullifier,
        created_at: env::block_timestamp(),
        delegator: "mpc.network.near".into(),  // MPC network!
        nonce: mpc_proof.nonce,
    }
    
    self.identities.insert(&commitment, &identity)
    self.nullifiers.insert(&nullifier)
    
    identity
}

// Verify MPC proof
fn verify_mpc_proof(proof: &MPCProof) -> bool {
    // 1. Check MPC network signature
    // 2. Verify threshold signatures (3 of 5 nodes signed)
    // 3. Check computation hash matches
    // 4. Verify timestamp is recent
    
    proof.verify_threshold_signature(3, 5)
        && proof.verify_computation_hash()
        && proof.is_recent()
}
```

### MPC Client Implementation

```typescript
import { MPCClient } from 'near-mpc-sdk'

class MPCDelegator {
  private mpcClient: MPCClient
  
  constructor() {
    this.mpcClient = new MPCClient({
      networkId: 'testnet',
      mpcContract: 'mpc.network.near',
      threshold: 3,
      totalNodes: 5
    })
  }
  
  async registerIdentity(
    accountId: string,
    npub: string,
    signature: string,
    publicKey: string,
    message: string
  ) {
    // 1. Secret-share the account_id
    const secretShared = await this.mpcClient.secretShare(accountId, {
      threshold: 3,
      totalNodes: 5
    })
    
    // 2. Define MPC computation
    const computation = {
      // Program to execute in MPC
      program: `
        // Step 1: Verify NEP-413 signature
        input signature, publicKey, message
        verify_nep413(signature, publicKey, message)
        
        // Step 2: Generate commitment
        input accountId (secret-shared)
        commitment = sha256("commitment:" || accountId)
        
        // Step 3: Generate nullifier
        nullifier = sha256("nullifier:" || accountId || nonce)
        
        // Output public results
        output commitment, nullifier
      `,
      
      // Private inputs (secret-shared)
      privateInputs: {
        accountId: secretShared
      },
      
      // Public inputs
      publicInputs: {
        npub,
        signature,
        publicKey,
        message
      }
    }
    
    // 3. Execute MPC computation
    const result = await this.mpcClient.execute(computation)
    
    // 4. Get MPC proof
    const mpcProof = result.proof
    
    // 5. Call smart contract
    const txResult = await this.mpcClient.callContract({
      contractId: 'nostr-identity.near',
      methodName: 'register_via_mpc',
      args: {
        npub,
        commitment: result.outputs.commitment,
        nullifier: result.outputs.nullifier,
        mpc_proof: mpcProof,
        mpc_signature: result.mpcSignature
      }
    })
    
    return {
      success: true,
      commitment: result.outputs.commitment
    }
  }
}
```

## 🔐 Security Guarantees

✅ **Threshold Cryptography**

- Single malicious node learns nothing

✅ **Zero-Knowledge Proofs**
- MPC proves computation was correct
- Without revealing private inputs

✅ **Verifiable Computation**
- Anyone can verify MPC executed correctly
- Cannot fake verification

### Privacy Guarantees

✅ **Information-Theoretic Privacy**
- Account_id split across nodes
- No single node sees full account
- Mathematically impossible to reconstruct without threshold

✅ **No Database**
- Nothing to leak
- No storage of account mappings
- Stateless verification

✅ **Forward Secrecy**
- Even if nodes compromised later
- Cannot decrypt past transactions

### Availability Guarantees

✅ **Decentralized**
- Multiple nodes across network
- No single point of failure

✅ **Censorship Resistant**
